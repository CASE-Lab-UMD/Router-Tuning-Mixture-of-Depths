Dynamic Depth for Transformers

Router-Tuning without per-model rewrites

Router-Tuning now loads standard Hugging Face causal language models and injects routing behavior at runtime. The result is a cleaner workflow: no `ckpt/*-mod` forks, no copied modeling files, and direct support for sequence-level and token-level routing.

What Changed

Model Loading

The training stack now patches compatible decoder layers at runtime, so standard `AutoModelForCausalLM` loading works for common backbones without local custom model files.

Routing Scope

Routing can target attention, MLP, or the whole block. Each mode also supports sequence-level and token-level gating.

Repository Cleanup

The old `ckpt/` forked checkpoints and duplicated modeling code are gone, making the repo smaller and easier to extend.

Training Workflow

Recommended Defaults

  • Base model: `Qwen/Qwen2.5-7B`
  • Granularity: `attn_sequence`
  • Router-only tuning enabled
  • `trust_remote_code=False` unless required

Supported Granularities

  • `attn_sequence`
  • `attn_token`
  • `mlp_sequence`
  • `mlp_token`
  • `block_sequence`
  • `block_token`
MODEL_NAME_OR_PATH="Qwen/Qwen2.5-7B" \
RUN_NAME="qwen2.5-7b" \
NUM_PROCESSES=4 PORT=29501 \
bash scripts/finetune_router_tuning.sh

Why This Layout Helps

Less Maintenance

Adding a new compatible model no longer means copying and editing a full `modeling_*.py` implementation.

Cleaner Experiments

Routing behavior is expressed in one patching layer, which makes it easier to compare targets and routing levels.
